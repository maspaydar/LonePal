Role: Senior Full-Stack IoT Engineer (Node.js/TypeScript + ESP32/C++).

Context: I am building a senior-care SaaS. My tech stack is Node.js/Express/TypeScript (Backend), Drizzle ORM/Postgres (DB), and React/Vite (Frontend). The hardware is an ESP32-S3-BOX-3 equipped with an HLK-LD2410 mmWave radar sensor.

Task: Create a bidirectional communication system and customization pipeline between the ESP32 and the SaaS.

1. Backend Integration (Node.js/TypeScript):

WebSocket Handler: Update the existing WebSocket server to handle specific event types from the ESP32 (e.g., SENSOR_DATA, DEVICE_STATUS).

Drizzle Schema: Update the database schema to include a device_settings table. It must store: sensitivity, detection_distance, ai_check_in_frequency, and active_hours.

Config API: Create a GET endpoint /api/devices/:mac_address/config that returns the resident's specific customization settings as JSON, signed with an HMAC-SHA256 signature for security.

2. Embedded Firmware (ESP32-S3-BOX-3 / C++):

HLK-LD2410 Driver: Write a module to interface with the mmWave radar via UART to detect presence and motion.

WebSocket Client: Implement a persistent WebSocket connection to the Node.js server.

Dynamic Customization: On startup, the ESP32 must fetch its JSON configuration from the SaaS API. The firmware must dynamically adjust the radar's sensitivity and reporting frequency based on this JSON without needing a re-flash.

Remote Diagnostic Tunnel: Implement the remote diagnostic capability as described in the architecture, using HMAC-SHA256 for signed command execution.

3. Frontend (Resident Web App - React):

Customization UI: Create a "Device Settings" component in the Resident Web App. Use TanStack Query to fetch current settings and a form to update them.

The "Upload" Action: When "Save to Device" is clicked, use the WebSocket server to send a CONFIG_UPDATE signal directly to the ESP32 to trigger an immediate settings refresh.

Important: Keep all backend code in TypeScript and ensure all hardware communication adheres to the existing security protocols (JWT/HMAC). Do not alter the core AI or authentication logic already present in the app.
